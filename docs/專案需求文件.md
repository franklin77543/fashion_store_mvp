# Fashion Store MVP - 專案需求文件

## 1. 專案概述

### 1.1 專案名稱
**Fashion Store MVP** - AI 驅動的時尚商品銷售平台

### 1.2 專案目標
建立一個現代化的電商平台，讓使用者能夠：
- 瀏覽並搜尋時尚商品
- 透過多種方式篩選商品（類別、顏色、價格、風格）
- 獲得個性化的商品推薦
- 使用自然語言描述需求（例如："我需要適合夏天穿的清爽洋裝"）
- 完成線上購物流程

### 1.3 專案背景
隨著電商市場的快速發展，使用者對於購物體驗的要求越來越高。本專案旨在：
- 提供直覺的商品瀏覽體驗
- 利用 AI 技術理解使用者的購物需求
- 透過開放資料集（Fashion Product Images Dataset）建立完整的商品資料庫
- 實踐現代化的全端開發架構（Clean Architecture）

## 2. 核心功能需求

### 2.1 商品瀏覽系統

#### 2.1.1 商品列表
- **功能描述**: 展示所有可購買的商品，支援分頁
- **輸入**: 頁碼、每頁數量
- **輸出**: 商品列表（圖片、名稱、價格、評分）
- **範例**:
  - 使用者進入首頁，看到第一頁 20 件商品
  - 使用者點擊第 2 頁，載入下一批 20 件商品

#### 2.1.2 商品詳情
- **功能描述**: 查看單一商品的完整資訊
- **輸入**: 商品 ID
- **輸出**: 商品詳細資訊（多張圖片、詳細描述、規格、庫存、評價）
- **範例**:
  - 使用者點擊商品卡片，進入詳情頁
  - 顯示商品的多角度圖片、材質、尺寸表、使用者評論

#### 2.1.3 商品搜尋
- **功能描述**: 使用關鍵字搜尋商品
- **輸入**: 搜尋關鍵字（中文/英文）
- **輸出**: 符合條件的商品列表
- **範例**:
  - 輸入 "洋裝" → 返回所有洋裝類商品
  - 輸入 "紅色" → 返回所有紅色系商品

#### 2.1.4 進階篩選
- **功能描述**: 多維度篩選商品
- **篩選條件**:
  - 性別（男性、女性、中性）
  - 類別（上衣、褲子、洋裝、外套、配件等）
  - 顏色（多選）
  - 價格範圍（滑桿選擇）
  - 季節（春、夏、秋、冬）
  - 品牌（下拉選單）
- **輸出**: 符合所有條件的商品列表

### 2.2 智能推薦系統（進階功能）

#### 2.2.1 自然語言搜尋
- **功能描述**: 使用自然語言描述購物需求
- **輸入**: 自然問句
- **輸出**: 推薦商品列表
- **範例**:
  - "我想找適合約會穿的優雅洋裝" → 推薦正式、優雅風格的洋裝
  - "夏天海邊度假穿什麼" → 推薦輕便、休閒的夏季服飾
  - "辦公室穿搭推薦" → 推薦商務、正式風格的服飾

#### 2.2.2 相似商品推薦
- **功能描述**: 根據當前瀏覽的商品，推薦相似商品
- **輸入**: 商品 ID
- **輸出**: 5-10 件相似商品
- **相似度計算**: 基於類別、顏色、風格、價格區間

#### 2.2.3 個性化推薦（可選）
- **功能描述**: 根據使用者瀏覽記錄和購買歷史推薦
- **輸入**: 使用者 ID、瀏覽歷史
- **輸出**: 個性化商品推薦

### 2.3 購物車系統

#### 2.3.1 加入購物車
- **功能描述**: 將商品加入購物車
- **輸入**: 商品 ID、數量、規格（尺寸、顏色）
- **輸出**: 購物車更新狀態

#### 2.3.2 購物車管理
- **功能描述**: 管理購物車內的商品
- **操作**:
  - 修改商品數量
  - 刪除商品
  - 清空購物車
  - 查看總金額

### 2.4 訂單系統（基礎版）

#### 2.4.1 結帳流程
- **功能描述**: 完成購物結帳
- **步驟**:
  1. 確認購物車商品
  2. 填寫收件資訊（姓名、地址、電話）
  3. 選擇付款方式（貨到付款、信用卡）
  4. 確認訂單並送出

#### 2.4.2 訂單查詢
- **功能描述**: 查看訂單狀態
- **輸入**: 訂單編號或使用者 ID
- **輸出**: 訂單詳情、配送狀態

## 3. 商品資訊展示需求

### 3.1 商品卡片（列表頁）
- **商品圖片**: 主要展示圖（懸停顯示第二張圖）
- **商品名稱**: 清晰易讀
- **價格**: 原價、折扣價（如有）
- **評分**: 星級評分 + 評論數量
- **標籤**: 熱銷、新品、折扣等標籤
- **快速操作**: 加入購物車按鈕、收藏按鈕

### 3.2 商品詳情頁
- **商品圖片**: 多角度圖片輪播（3-5 張）
- **基本資訊**:
  - 商品名稱
  - 品牌
  - 價格（原價、折扣價）
  - 評分與評論數
- **規格選擇**:
  - 顏色選擇器（色塊展示）
  - 尺寸選擇（S/M/L/XL）
  - 數量選擇器
- **詳細說明**:
  - 商品描述
  - 材質成分
  - 尺寸表
  - 洗滌說明
- **使用者評論**: 評分、評論內容、評論者、日期
- **相關推薦**: 相似商品、搭配推薦

## 4. 使用者介面需求

### 4.1 首頁
- **導覽列**: Logo、搜尋框、購物車、使用者中心
- **輪播廣告**: 促銷活動、新品上市
- **分類導航**: 快速進入各類別
- **精選商品**: 熱銷商品、新品推薦
- **主題專區**: 季節推薦、風格專區

### 4.2 商品列表頁
- **篩選側邊欄**: 多維度篩選條件
- **排序選項**: 價格、熱銷度、新品、評分
- **商品網格**: 響應式網格佈局（3-4 欄）
- **分頁控制**: 頁碼、上下頁

### 4.3 商品詳情頁
- **左側**: 商品圖片展示區
- **右側**: 商品資訊、規格選擇、購買按鈕
- **下方**: 詳細說明、使用者評論、相關推薦

- **商品列表**: 商品圖片、名稱、規格、單價、數量、小計
- **總計區域**: 商品總數、總金額、結帳按鈕
- **優惠碼**: 輸入優惠碼欄位

### 4.5 結帳頁面
- **步驟指示**: 顯示當前步驟（購物車 → 填寫資訊 → 確認訂單 → 完成）
- **資訊表單**: 收件人、地址、電話、付款方式
- **訂單摘要**: 商品清單、總金額

## 5. 技術需求

### 5.1 資料來源
- **資料集**: Fashion Product Images Dataset (Kaggle)
  - 來源: https://www.kaggle.com/datasets/paramaggarwal/fashion-product-images-dataset
  - 總數: 約 44,448 件商品資料
  - 位置: `fashion-dataset/`

#### 資料檔案結構
```
fashion-dataset/
├── styles.csv                 # 商品基本資訊 (44,448 筆)
├── images.csv                 # 商品圖片清單與 URL
├── images/                    # 商品圖片目錄 (JPG 格式)
└── styles/                    # 商品詳細 JSON 資料
    └── [10000-60000].json    # 每個商品的完整資訊
```

#### styles.csv 欄位
- **id**: 商品唯一識別碼
- **gender**: 性別分類 (Men/Women/Boys/Girls/Unisex)
- **masterCategory**: 主要類別 (Apparel/Accessories/Footwear)
- **subCategory**: 子類別 (Topwear/Bottomwear/Watches/Shoes/Bags等)
- **articleType**: 商品類型 (Shirts/Jeans/Watches/Casual Shoes/Handbags等)
- **baseColour**: 基礎顏色 (Black/White/Blue/Red等)
- **season**: 適用季節 (Summer/Winter/Fall/Spring)
- **year**: 年份 (2011-2016)
- **usage**: 使用場合 (Casual/Formal/Ethnic/Sports/Party等)
- **productDisplayName**: 商品完整顯示名稱

#### styles/[id].json 詳細資訊
- **價格資訊**: price, discountedPrice, discountPercent
- **品牌資訊**: brandName, brandUserProfile
- **商品詳情**: productDescriptors (描述、材質、洗滌說明)
- **圖片資源**: styleImages (default/front/back/search 等多角度)
- **尺寸選項**: styleOptions (可用尺寸與庫存)
- **分類資訊**: masterCategory, subCategory, articleType
- **屬性標籤**: articleAttributes (Pattern/Fit Type等)

#### 圖片處理
- **格式**: JPG
- **多角度**: 支援 default/front/back/search 等多張圖片
- **多解析度**: 48x64, 125x161, 180x240, 360x480, 1080x1440
- **位置**: 本地 `fashion-dataset/images/` 目錄

#### 資料清洗需求
- 處理缺失值 (部分欄位可能為空)
- 標準化顏色名稱 (統一命名規則)
- 驗證圖片檔案存在性
- 價格資料驗證與格式化
- 處理重複資料

### 5.2 搜尋與推薦引擎
- **關鍵字搜尋**: 
  - 基於 SQLAlchemy LIKE 查詢
  - 支援中文分詞（可選）
- **語義搜尋**（進階）:
  - Sentence Transformers（多語言模型）
  - 向量相似度計算
  - ChromaDB 或 Pickle 向量儲存
- **推薦演算法**:
  - 基於內容的推薦（相似商品）
  - 協同過濾（可選，需使用者行為數據）

### 5.3 AI 自然語言理解（進階功能）
- **LLM 整合**: Ollama (Llama 3.1) 或 OpenAI API
- **意圖識別**: 解析使用者需求（風格、場合、季節、顏色）
- **實體提取**: 提取關鍵屬性（正式/休閒、顏色偏好、價格範圍）

## 6. 非功能性需求

### 6.1 效能需求
- **頁面載入時間**: 首頁 < 2 秒
- **API 回應時間**: 商品列表 < 500ms，搜尋 < 1 秒
- **圖片載入**: 使用懶加載（Lazy Loading）
- **快取策略**: 商品列表快取 5 分鐘

### 6.2 使用者體驗
- **響應式設計**: 支援桌面、平板、手機
- **無障礙設計**: ARIA 標籤、鍵盤導航
- **載入狀態**: Skeleton Loading、載入動畫
- **錯誤處理**: 友善的錯誤提示訊息

### 6.3 安全性
- **輸入驗證**: 防止 SQL Injection、XSS 攻擊
- **HTTPS**: 所有 API 使用 HTTPS（生產環境）
- **使用者認證**: JWT Token 認證（如需登入功能）
- **資料加密**: 敏感資料加密儲存

### 6.4 擴展性
- **模組化設計**: Clean Architecture（Repository → Service → API）
- **資料庫設計**: 正規化設計，易於擴展
- **API 版本控制**: 支援 API 版本升級
- **微服務準備**: 可拆分為獨立服務（商品、訂單、使用者）

## 7. 專案範圍

### 7.1 第一階段（MVP）
1. 商品資料庫建立（Fashion Dataset 匯入）
2. 商品列表與詳情頁展示
3. 基礎搜尋功能（關鍵字）
4. 分類篩選（性別、類別、顏色、價格）
5. 購物車系統（前端狀態管理）
6. 簡易結帳流程（不含真實付款）
7. 響應式 Web 介面

### 7.2 第二階段（進階功能）
1. AI 自然語言搜尋（Ollama LLM）
2. 語義搜尋（向量相似度）
3. 相似商品推薦
4. 使用者註冊/登入系統
5. 訂單管理與查詢
6. 使用者評論功能
7. 收藏清單

### 7.3 未來擴展
- 以圖搜圖功能（圖片相似度比對）
- 虛擬試穿（AR 技術）
- 個性化推薦系統
- 即時庫存管理
- 真實金流串接（綠界、藍新）
- 物流追蹤整合
- 多語言支援（繁中、簡中、英文）
- 行動 App（React Native）
